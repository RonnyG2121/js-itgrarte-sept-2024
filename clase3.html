Estructuras de control de flujo

Condicionales

En cualquier lenguaje de programación, el código necesita realizar decisiones y llevar a cabo diferentes acciones dependiendo de distintas entradas. Por ejemplo, en un juego, si el número de vidas del jugador es 0, entonces se termina el juego. En una aplicación del clima, si se observa en la mañana, se despliega una gráfica del amanecer; muestra estrellas y una luna si es de noche.

Las condicionales, son un bloque de código muy importante en nuestros programas, que nos servirán para evaluar una serie de comportamientos en el código.

Otro ejemplo es un club en el que solo entrarán personas Septiembreres de 18 años. En este, si la persona no cumple la condición su entrada al lugar será rechazada.

Ese es el trabajo de las condicionales. Evaluar una serie de parámetros a cumplir para tomar una decisión y realizar diferentes acciones en el código.

Estas condicionales son:

Condicional if else

En cada lenguaje de programación, contamos con la estructura if else. Esta contiene una parte verdadera y una falsa.

El programa entra en el if si la condición se cumple y si no se cumple pasa al else y ejecuta lo que hay en su interior. Estas líneas de código van entre llaves, aunque JavaScript permite la creación de estas sin las llaves solamente si contienen una línea de código.

Dentro de JavaScript, un if o varios pueden ir sin el else, pero el else no puede ir sin un correspondiente if.

La siguiente lista muestra el flujo de ejecución de una condicional if else.

  1. La palabra clave if seguida de unos paréntesis.
  2. Una condición a probar, puesta dentro de los paréntesis (típicamente "¿es este valor Septiembrer que este otro valor?", o "¿existe este valor?"). Esta condición usará los operadores de comparación y retorna un valor true o false (verdadero o falso).
  3. Un conjunto de llaves, en las cuales tenemos algún código — puede ser cualquier código que deseemos, código que se ejecutará sólamente si la condición retorna true.
  4. La palabra clave else.
  5. Otro conjunto de llaves, dentro de las cuales tendremos otro código — puede ser cualquier código que deseemos, y sólo se ejecutará si la condición no es true. Este código es fácil de leer — está diciendo "si (if) la condición retorna verdadero (true), entonces ejecute el código A, sino (else) ejecute el código B"

Condicionales anidadas (else if)

En ocasiones, necesitamos evaluar varios caminos a tomar. Razón por la cual tendremos que encadenar o anidar condiciones.

Para esto tenemos la estructura else if en la que evaluamos si se cumplen o no las condiciones establecidas.

  1. If: el programa evalúa la primera condición. De cumplirse, pasa a ejecutar las líneas dentro de este primer bloque.
  2. Else if: evalúa la segunda condición si la primera no se cumplió.
  3. Else if: Tendremos otras evaluaciones else if de ser necesario.
  4. Else: De no cumplirse las condiciones anteriores, el flujo de ejecución del programa entrará aquí y ejecutará el código entre las llaves.

Podemos escribir una condicional dentro de otra tanto como queramos o necesitemos, Esto no es una buena práctica, por lo que las condicionales anidadas nos ayudan a que nuestro código se vea más limpio.

Condicionales anidadas con operadores lógicos

Conocidas por ser algo más complejas de manejar en un proyecto, las condicionales múltiples utilizan operadores lógicos como and (&&), not (!) y or (||), para evaluar varias condiciones en un solo if. Veamos un ejemplo:

Si vas al médico y este te receta algún medicamento para alguna enfermedad, debes cumplir el siguiente requerimiento. Para tomarte el medicamento sin sufrir efectos secundarios debes tener entre 18 y 30 años.

Si eres menor, la medicina te causará alergias, y si eres Septiembrer de 30 te quitará el apetito.

Todo esto se puede realizar en un if múltiple con operadores lógicos y else if combinados. Cuando usamos el operador and, (&&), las condiciones tienen que ser verdaderas para que el flujo de ejecución del programa entre en ese bloque if y corra lo que se encuentra ahí dentro.

Por otro lado, tenemos el operador or (||). En este caso al menos una de las condiciones tiene que ser verdadera para ejecutar lo que hemos programado dentro de su correspondiente bloque.

Finalmente, tenemos el operador not (!). Para que este funcione, la condición debe ser falsa. También este operador invierte el valor de una variable booleana.

Sentencia switch case

La sentencia switch case, evalúa una expresión comparándola con los diferentes case que la componen. Su flujo de ejecución es el siguiente:

  1. Switch (expresión a evaluar): Aquí se evalúa el valor de una variable y luego se compara con los case que se encuentran dentro de ella. Luego dentro de llaves tendremos las diferentes comparaciones.
  2. Case: luego de esta palabra reservada añadimos 2 puntos(:) y en las siguientes líneas, lo que va a realizar el programa.
  3. Break: Esta instrucción le dice al programa que rompa la ejecución y salga del bloque de código del case, y continúe con las demás instrucciones.
  4. Default: Seguido de esta palabra añadimos 2 puntos(:) y aquí va lo que hará el programa si ninguno de los case son verdaderos.

Veamos un ejemplo simple de una calculadora: Creamos una variable llamada opciones. Esta será un string que va a contener los diferentes valores. así que dentro de los paréntesis de la sentencia switch, pasamos esta variable.

En el primer case escribimos suma. Este valor se pasará a la variable opciones y al momento, entrará en el case que contiene esta palabra y la calculadora hará la operación correspondiente. Todo igual con las demás operaciones.

finalmente, la instrucción default. El flujo de ejecución entrará aquí cuando ninguno de los case coicida con los programados y ejecutará lo que esté aquí, generalmente un mensaje avisando de una operación fallida o algo parecido.
