Funciones

Las funciones, son un bloque de código que devuelve un resultado al ser llamadas. Son conocidas como ciudadanos de primera clase, ya que en Javascript se puede hacer prácticamente de todo con ellas.

Las funciones son uno de los bloques de construcción fundamentales en JavaScript. Una función en JavaScript es similar a un procedimiento — un conjunto de instrucciones que realiza una tarea o calcula un valor. Para usar una función, debes definirla en algún lugar del ámbito desde el que deseas llamarla.

En JavaScript, las funciones también son un tipo de dato por increíble que parezca. Son fundamentales en la reutilización de código, para que no tengamos que repetir líneas de código en todo el programa.

Luego de definir la función, debemos llamarla para que esta realice su procedimiento. de modo que, no es solo definirla, debemos llamarla en cualquier parte del programa y de esta manera reutilizamos un fragmento de código siempre que lo necesitemos.

Ámbito de variables en funciones

Dentro de las funciones podemos declarar variables, pero al momento de utilizar esas variables, no podemos modificarla fuera del bloque de código que conforma la función, que dicho sea de paso va entre llaves. De igual manera, si declaramos una variable global y la modificamos dentro de la función, su valor cambiará dentro de ella y seguirá teniendo su valor predefinido en el ámbito en que fue declarada.

Declaración de una función

Podemos declarar una función de distintas formas. Y una de ellas, o la principal, es usando la palabra reservada function, luego el nombre y este tiene que ser sin espacios, sin caracteres especiales y puede ser con guiones bajos. Seguido del nombre, los paréntesis o zona de argumentos. Finalmente el código que tendrá entre llaves.

Otra forma es declararla como una expresión, es decir, como si fuera una variable. Podemos declararla como una función anónima, que se ejecutará sin llamarla.

Ejecución de una función

Una función, a menos que esta sea anónima, debe ser llamada para ejecutarse. Como se había explicado líneas más arriba, no solo es crearla, también debemos llamarla donde queramos que haga su procedimiento.

Si creamos una función que nos calcule el área de un rectángulo, debemos llamarla más adelante y pasarle los datos correspondientes para que esta nos devuelva el resultado.

Zona de parámetros

Dentro de los paréntesis de una función tenemos lo que conocemos como zona de parámetros o zona de argumentos. Aquí el programador pasa los valores que luego se utilizarán dentro.

Por ejemplo: Una función que realiza una suma de números recibe 2 parámetros, num1 y num2. en el return de la función, haremos la operación de la suma y al llamarla, le pasamos los valores por ejemplo 2, 2. Estos parámetros se ceparan por una coma cada uno.

Instrucción return

Puede que una función devuelva un valor cuando la llamemos en el programa, o puede que no dependiendo lo que estemos haciendo. Si queremos que devuelva un valor, (como el resultado de una suma) debemos usar la instrucción return que significa devolver. Luego, el desarrollador decidirá que hará con el valor ya devuelto por la función.

Funciones flecha

Las funciones flecha, o (Arrow function) son una nueva forma de compactar el código en JavaScript, solo que con algunas diferencias con las normales. La forma de declararlas es la siguiente: palabra reservada const o let, (ya que la haremos como una expresión) seguido de esto el nombre y luego el signo igual (=). Luego del signo, tendremos la zona de los parámetros en el que si solo existe uno, podemos omitir los paréntesis. Después, el signo de la flecha gorda o (=>) y finalmente las llaves.

En este tipo de funciones tenemos algunas diferencias para reducir código:

  • Podemos escribir en una sola línea si la función es pequeña y no llevará más de una línea de código.
  • Podemos omiti la palabra return si la función tiene una sola línea.
  • Podemos omitir los paréntesis si solo tenemos un parámetro en la función.
